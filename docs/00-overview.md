# 00. 프로젝트 개요

## 1) 배경/목표
**뚱퀀스(DdungQuence)**는 “만날 때마다 시퀀스 보드게임을 즐기는 친구들(뚱팸)”이  
**온라인에서도 빠르고 편하게** 플레이하기 위해 시작한 프로젝트다.

- 제품/서비스명: **뚱퀀스(DdungQuence)**
- 게임 장르: 칩 배치 기반 **5-in-a-row(시퀀스 계열) 턴제 보드게임**
- 목표: 서버 없이(또는 최소 운영) **Firebase 무료 구간 중심**으로 실시간 멀티플레이 제공

### 기술 스택
- 프론트: **Next.js(App Router) + TypeScript + TailwindCSS**
- 실시간 동기화: **Firebase Firestore(무료 구간 중심 설계)**
- 인증: **Firebase Auth** — 현재 구현은 익명 인증(Anonymous), 목표는 Google 로그인
- 배포: **Vercel**

## 2) 성공 기준(Outcome)
- 4명이 방에 들어와 **10초 내 게임 시작**
- 1턴 처리(내 액션 → 상대 반영)가 **1초대(체감)**로 동작
- 모바일에서 보드 조작(탭/드래그/확대)이 불편하지 않음
- 운영 비용: 무료 구간(또는 소액) 유지

## 3) 범위(Scope)

### 포함(MVP)
- 방 생성/참가(코드 기반)
- 로비(팀 선택/Ready/시작)
- 게임 플레이(턴 진행/카드 선택/칩 놓기/잭 특수)
- 승리 판정(시퀀스 완성)
- 재접속/동기화(네트워크 불안정 기본 대응)
- 리플레이/관전(최대 2명)/랭킹

### 제외(초기)
- 결제/광고/수익화
- 대규모 매칭(랜덤 매칭)

## 4) 원칙(개발/운영)
- **서버리스 우선**: 가능한 모든 로직은 클라이언트 + Firestore 트랜잭션으로 해결
- **턴 기반 이벤트 최소화**: “전체 상태 스냅샷” 남발 금지(읽기/쓰기 비용 폭증 방지)
- **저작권/상표 리스크 관리**: 원작과 동일한 명칭/아트워크/룰북 문구를 그대로 재사용하지 않는다(특히 공개 배포 시)

## 5) 용어(프로젝트 내부)
- Turn: 한 플레이어의 액션 1회(카드 1장 사용 + 칩 배치/제거)
- Sequence: 같은 팀 칩 5개 연속(가로/세로/대각)
- 팀 색상: 레드(1번 팀) · 블루(2번 팀) · 그린(3번 팀), 배정 순서대로 적용
- One-eyed Jack: 제거(상대 칩 1개 제거, 단 완성된 시퀀스 일부는 제거 불가)
- Two-eyed Jack: 와일드(원하는 곳에 배치)
